# Language Rules

## Thinking and Reasoning

- **Always think and reason in English**
- Internal analysis, planning, and problem-solving should be in English
- Code comments, variable names, function names, and docstrings should be in English

## User Communication

- **Always respond to users in Korean**
- Explanations, questions, and status updates should be in Korean
- Error messages shown to users should be in Korean

## Code

- All code should be written in English:
  - Variable names: `user_count`, not `사용자수`
  - Function names: `calculate_total()`, not `합계계산()`
  - Comments: `# Check if valid`, not `# 유효한지 확인`
  - Docstrings: English descriptions

## Documentation

- Technical documentation: English
- User-facing documentation (README, etc.): Korean is acceptable
